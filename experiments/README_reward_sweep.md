# Reward Shaping Sweep

## What it does

Trains single-song PPO policies across multiple songs and reward shaping variants, then produces a comparison report.

## Default setup

- **Songs (recommended for harder/longer evaluation):**
  - `Pirates` â€” 287s, 5758 steps (long, complex multi-hand piece)
  - `Hello` â€” 291s, 5826 steps (longer duration)
  - `Beginners` â€” 298s, 5959 steps (longest available)
  - `TwinkleTwinkleRousseau` â€” 7.5s, 150 steps (quick sanity check)
  - Default eval uses: `TwinkleTwinkleRousseau`, `Pirates_1`, `Stan_1`, `Petrunko_3`
- **Variants (8 total):**
  - `baseline` â€” original PianoMime reward (no extra shaping)
  - `vel_smooth` â€” penalize joint velocity jerk
  - `action_smooth` â€” penalize large action changes
  - `preposition` â€” reward finger approach before note onset
  - `timing` â€” reward precise note onset timing
  - `finger_dist` â€” reward finger-key proximity
  - `collision` â€” penalize hand-hand fingertip collisions
  - `smooth_combo` â€” velocity + action smoothness
  - `full_combo` â€” all shaping rewards combined
- **Output root:** `/tmp/pianomime_reward_sweep`

## One-command run (local, 1 GPU)

```bash
bash pianomime/experiments/run_reward_sweep.sh
```

## Quick smoke test (dry run)

```bash
python pianomime/experiments/reward_shaping_sweep.py \
    --dry-run \
    --songs TwinkleTwinkleRousseau Pirates_1 \
    --variants baseline smooth_combo \
    --cuda-devices 0 1 2 3
```

## Outputs

Each training run produces:

- `<root>/<run>/args.json` â€” experiment hyperparameters
- `<root>/<run>/result.json` â€” final metrics (F1, precision, recall, reward, etc.)
- `<root>/<run>/eval_history.jsonl` â€” per-iteration eval metrics (one JSON per line)
- `<root>/<run>/eval_best.mp4` â€” video from the iteration with the highest F1
- `<root>/<run>/eval_final.mp4` â€” video from the final evaluation

The sweep aggregator produces:

- `<root>/summary.json` â€” machine-readable results
- `<root>/summary.md` â€” human-readable comparison table
- `<root>/training_curves.png` â€” F1 training curves comparing all variants across songs

## Command-line reference

| Flag                  | Default  | Description                      |
| --------------------- | -------- | -------------------------------- |
| `--songs`           | 4 songs  | Songs to evaluate                |
| `--variants`        | all 9    | Reward variants to compare       |
| `--seeds`           | `42`   | Random seeds                     |
| `--total-iters`     | `300`  | Training iterations per run      |
| `--num-envs`        | `8`    | Parallel envs per training       |
| `--n-steps`         | `512`  | PPO n_steps                      |
| `--residual-factor` | `0.03` | Residual scaling factor          |
| `--cuda-devices`    | `0`    | GPU IDs for parallel execution   |
| `--sequential`      | off      | Run one job at a time            |
| `--dry-run`         | off      | Print commands without executing |

## Suggested experiment order

1. **Smoke test:** `baseline` + `smooth_combo` on 1 song
2. **Ablation:** Run all single-reward variants (`vel_smooth`, `action_smooth`, `preposition`, `timing`, `finger_dist`, `collision`)
3. **Combos:** `smooth_combo` + `full_combo`
4. **Multi-seed:** Add `--seeds 42 43 44` for statistical significance
5. **Scale:** Increase `--total-iters` to 500â€“1000 for final results

## Notes

- All shaping rewards are **opt-in**; the baseline preserves original behavior.
- The launcher uses FIFO scheduling across GPUs â€” each GPU runs one job at a time.
- Songs must have corresponding `_left_hand_action_list.npy` and `_right_hand_action_list.npy` files under `<dataset-root>/high_level_trajectories/`.
- Training checkpoints and tensorboard logs are written to `experiment_dir/` by train_ppo.py.

## Final Deliverables

After running the sweep, you get everything needed for the report:

| Deliverable                             | How to obtain                                        |
| --------------------------------------- | ---------------------------------------------------- |
| **F1 / metrics table**            | `summary.md` and `summary.json`                  |
| **Training curve plot**           | `training_curves.png` (auto-generated by sweep)    |
| **Play-through video (baseline)** | `<run>/eval_best.mp4` for `baseline` variant     |
| **Play-through video (improved)** | `<run>/eval_best.mp4` for best-performing variant  |
| **Training log**                  | `tensorboard --logdir robopianist_rl/tensorboard/` |

For a quick single-run report:

```bash
# Train one baseline run
python pianomime/single_task/train_ppo.py \
    --mimic-task Pirates --total-iters 500 --residual-action --gravity-compensation \
    --num-envs 8 --name baseline__Pirates

# Plot its training curve
python pianomime/experiments/plot_training_curves.py \
    --exp-dir /tmp/pianomime_reward_sweep/baseline__Pirates \
    --output baseline_pirates_curve.png

# Train the improved version
python pianomime/single_task/train_ppo.py \
    --mimic-task Pirates --total-iters 500 --residual-action --gravity-compensation \
    --num-envs 8 --name smooth_combo__Pirates \
    --reward_velocity_smoothness --reward_velocity_smoothness_coef 0.01 \
    --reward_action_smoothness --reward_action_smoothness_coef 0.01

# Compare both
python pianomime/experiments/plot_training_curves.py \
    --runs baseline__Pirates smooth_combo__Pirates \
    --root-dir /tmp/pianomime_reward_sweep \
    --output comparison.png
```
